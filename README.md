## 📄 Description des documents

### 🗂️ Documents principaux

| Fichier | Description |
|--------|-------------|
| `Cartographie des composants.jpg` | Vue d'ensemble visuelle de l'architecture et des composants du projet |
| `Flowly_Cahier_des_Charges.pdf` | Cahier des charges complet du projet Flowly — objectifs, contraintes, périmètre |
| `Flowly_Fonctionnalites.pdf` | Spécifications fonctionnelles détaillées des fonctionnalités du projet |
| `Flowly_PBS_WBS.pdf` | PBS (Product Breakdown Structure) et WBS (Work Breakdown Structure) — décomposition des livrables et des tâches |
| `OBS.png` | OBS (Organizational Breakdown Structure) — organigramme de l'équipe projet |

### 🔬 Veille Technologique

Études comparatives des technologies envisagées pour chaque couche du projet :

| Fichier | Domaine |
|--------|---------|
| `veille_backend_final.pdf` | Technologies back-end (serveurs, API, bases de données, etc.) |
| `Veille Techno Dev.pdf` | Outils et pratiques de développement |
| `veille_techno_front.pdf` | Frameworks et librairies front-end |
| `Veille technos Mob.pdf` | Solutions de développement mobile |

---

## 🚀 Utilisation

Ce dépôt est un dépôt de **documentation uniquement**. Il ne contient pas de code source.

Consultez les documents dans l'ordre suivant pour une bonne prise en main du projet :

1. `Cartographie des composants.jpg` — vue globale du système
2. `Flowly_Cahier_des_Charges.pdf` — contexte et objectifs
3. `Flowly_Fonctionnalites.pdf` — détail des fonctionnalités
4. `Flowly_PBS_WBS.pdf` — organisation des livrables
5. `OBS.png` — structure de l'équipe
6. Dossier `Veille technologique/` — choix techniques argumentés
